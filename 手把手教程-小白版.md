## 第1步：在你的电脑上，把项目上传到 GitHub

### 1.1 下载 Git

**做什么**：安装 Git 工具，让你能和 GitHub 交互

**在哪里操作**：你的电脑

**怎么做**：
1. 打开浏览器，访问 https://git-scm.com/download/win
2. 下载会自动开始（下载一个 exe 文件）
3. 双击安装，全部点"下一步"（Next）即可，不用改设置
4. 安装完成后，**重启电脑**

**验证安装**：
1. 在键盘上按 `Win + R`
2. 输入 `cmd`，按回车
3. 在弹出的黑色窗口中输入：
   ```
   git --version
   ```
4. 如果显示类似 `git version 2.x.x`，说明安装成功
5. **关掉这个黑色窗口**

---

### 1.2 在 GitHub 创建仓库

**做什么**：在 GitHub 上创建一个"文件夹"，用来存放你的代码

**在哪里操作**：你的电脑（浏览器）

**怎么做**：
1. 打开浏览器，访问 https://github.com
2. 登录你的账号
3. 点击右上角的 `+` 号，选择 `New repository`

4. 填写信息：

   - **Description**：输入 `Cosens 工业设备网站`
   - **Public**：选中（公开仓库）
   - **Add a README file**：**不要勾选**
   - **Add .gitignore**：**不要勾选**
   - **Choose a license**：**不要勾选**

5. 点击绿色的 `Create repository` 按钮

6. 创建完成后，你会看到一个页面，上面有个地址：
   ```
   https://github.com/你的用户名/cosens-website.git
   ```
   **复制这个地址，保存到记事本里，后面要用**

## 第2步：在云服务器上，生成 SSH 密钥

**做什么**：让 GitHub 能安全地连接到你的云服务器

**在哪里操作**：云服务器（不是你自己的电脑！）

**怎么做**：

### 2.1 连接到云服务器

**在哪里操作**：你的电脑

**怎么做**：

#### 方法一：使用 Windows 自带的 SSH（推荐）

1. 按 `Win + R`
2. 输入 `cmd`，按回车
3. 在黑色窗口中输入：
   ```bash
   ssh root@1.14.163.66
   ```
4. 按回车

5. 第一次连接会提示：
   ```
   Are you sure you want to continue connecting (yes/no/[fingerprint])?
   ```
   输入 `yes`，按回车

6. 提示输入密码：
   ```
   root@1.14.163.66's password:
   ```
   输入：`Sunan6668888.`
   
   > ⚠️ **注意**：输入密码时不会显示任何字符，这是正常的！

7. 如果成功，你会看到类似：
   ```
   Welcome to Ubuntu 22.04 LTS
   root@server:~#
   ```
   
   看到 `#` 就表示你已经连接到服务器了！

#### 方法二：如果上面的方法不行，使用 PuTTY

1. 下载 PuTTY：https://www.putty.org/
2. 安装后打开 PuTTY
3. 在 `Host Name` 输入：`1.14.163.66`
4. 点击 `Open`
5. 登录账号：`root`
6. 密码：`Sunan6668888.`

---

### 2.2 在云服务器上生成密钥

**在哪里操作**：云服务器（你已经连接上了）

**怎么做**：

**一次执行一行**，每行执行完按回车：

```bash
ssh-keygen -t ed25519 -C "github-actions@cosens.cn" -f ~/.ssh/github_actions -N ""
```
按回车

**你会看到**：
```
Generating public/private ed25519 key pair.
Your identification has been saved in /root/.ssh/github_actions
Your public key has been saved in /root/.ssh/github_actions.pub
```
这说明成功了！

> 💡 **说明**：这个命令生成了两把"钥匙"：
> - `github_actions.pub` = 公钥（放在服务器上）
> - `github_actions` = 私钥（给 GitHub 用）

---

### 2.3 把公钥添加到授权列表

**一次执行一行**：

```bash
cat ~/.ssh/github_actions.pub >> ~/.ssh/authorized_keys
```
按回车

> 💡 **说明**：这一步是让服务器"信任"这把钥匙

---

### 2.4 查看并复制私钥

**一次执行一行**：

```bash
cat ~/.ssh/github_actions
```
按回车

**你会看到一大串内容**，类似：
```
-----BEGIN OPENSSH PRIVATE KEY-----
abc123def456...
...
...
-----END OPENSSH PRIVATE KEY-----
```

**你需要**：
1. **选中并复制全部内容**（包括 `-----BEGIN` 和 `-----END` 这两行）
2. 粘贴到记事本里保存
3. **不要关掉服务器窗口**，等下还要用



> ⚠️ **重要安全提醒**：
> - 这个私钥相当于"钥匙"
> - **不要分享给任何人**
> - **不要上传到公开的地方**
> - 我们等下会把它放到 GitHub 的安全仓库里

**第2步完成！** ✅

## 🐛 常见问题

### 问题1：git push 提示 "failed to push"

**原因**：可能是仓库有冲突

**解决**：
```bash
git pull origin main --rebase
git push origin main
```

---

### 问题2：GitHub Actions 显示红色 ❌

**原因**：部署失败

**解决**：
1. 点击失败的 workflow
2. 查看红色的步骤
3. 看错误信息
4. 常见原因：
   - SSH 密钥不正确 → 重新生成并复制
   - 服务器连接不上 → 检查 IP 和端口
   - 权限问题 → 在服务器执行 `chmod 600 ~/.ssh/authorized_keys`

---

### 问题3：部署成功但网站没变化

**解决**：
```bash
# 在云服务器上执行
ssh root@1.14.163.66
cd /var/www/cosens
php artisan optimize:clear
sudo systemctl reload nginx
```
